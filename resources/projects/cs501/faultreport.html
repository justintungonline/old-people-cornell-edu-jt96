Found Faults

  • PDFs examples of malformed text:
    http://lula.law.cornell.edu/uscode/pdf/23/ch2.html
    http://lula.law.cornell.edu/uscode/pdf/23/506.html

    Kohsuke has suggested these problems with overlapping text can be corrected; however, the structural difficulties of the code the due to the status of FOP with some unsupported functions may cause PDFs to be displayed incorrectly (i.e. not reflecting the HTML version).

  • Occasional OFE Failure to connect to mail server:
    Possibly resulting from problems with the mail server.
  • Incorrect links on US code index page on lula:
    Index page of the current website may need correct links reflecting that the site is now on lula and not www4
  • Unreadable Mail:
    During testing, we found that mail sent to users does not function on hotmail accounts because both the text and the HTML attachment do now show up resulting in an empty message. However, while testing we did not discover any other email readers that exhibited this problem. One suspicion is that hotmail is very sensitive to attachments and perhaps changing the format of the mixed text will solve the problem.
